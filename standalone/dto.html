How this maps to the Croquet/Multisynq idea. The engine below is a deterministic model (a seeded simulation) and the page is a pure view of it — exactly Croquet's architecture. The only thing removed is the network reflector: this runs entirely on your desktop, offline, with no API key. Opening multiple windows shares the active scenario over the browser's local BroadcastChannel — a same-machine stand-in for the reflector. For guaranteed cross-window sync, serve the folder with python3 -m http.server and open http://localhost:8000/dto.html; double-clicking the file also works (sync is best-effort under file://).
